# 维护者

本页定义 OpenClaw.NET 的维护者角色和审查期望。它不列出现任维护者，也不通过赞助、雇佣或商业关系授予权限。

维护者身份基于持续贡献、信任、技术判断、审查可靠性和社区行为。

## 当前结构

### 核心维护者

核心维护者负责项目范围的技术方向和对以下变更的最终批准：

- 运行时架构
- 安全敏感行为
- 发布权限
- 包和项目边界
- 默认 Provider 行为
- 公开路线图决策
- 跨子系统兼容性

运行时/核心、网关安全、发布和包边界变更需要核心维护者批准。

### 区域维护者

区域维护者指导项目的限定范围区域。他们可以审查 Pull Request、维护文档、塑造示例，并在其范围内协调整贡献者工作。

区域维护者的权限是有范围限制的。核心运行时、安全敏感、发布和包边界变更仍然需要核心维护者批准。

### 审查者 / 受信任贡献者

审查者和受信任贡献者帮助评估正确性、测试覆盖、文档清晰度、兼容性和范围。他们可以提供有力的审查意见，但不持有维护者权限。

当配合持续贡献、良好的判断力和健康的社区行为时，受信任贡献者身份可以成为通向区域维护者身份的路径。

## 区域维护者范围

区域维护者的范围应当是明确的。范围可以包括文档区域、示例系列、适配器组、区域社区工作或扩展包。

示例区域：

| 区域 | 范围 | 边界 |
| --- | --- | --- |
| 中国社区与行业扩展 | 中文文档、社区反馈、行业示例、可复用的行业适配器指南 | 不授予对核心运行时、安全、发布、包边界、供应商默认配置或产品特定工作流的权限 |

当项目有持续的贡献者和明确的归属需求时，可以添加更多区域。

## 审查期望

维护者和审查者应检查：

- 与声明行为的正确性
- 足够的测试或明确的纯文档变更理由
- NativeAOT 兼容性和裁剪影响
- 安全态势和不安全操作的门控
- 行为、设置或兼容性变更的文档更新
- 范围控制和包边界纪律
- 不支持的模式的快速失败行为
- 行业或商业贡献的供应商中立性
- 贡献直接支持公司或客户用例时的披露

## 批准边界

区域维护者可以在其范围内批准或指导工作，前提是变更不影响核心运行时架构、网关安全、发布权限或包边界。

以下情况需要核心维护者批准：

- 变更核心运行时契约或行为
- 变更网关安全态势或公开绑定保护
- 变更发布流程、包输出或受支持的构建产物边界
- 变更默认 Provider 行为
- 添加没有明确扩展边界的重量级可选集成
- 在 AOT 敏感路径中引入反射、动态加载或隐藏的 JIT 依赖
- 创建新的项目方向或路线图承诺

有疑问时，审查者应询问该工作是否属于 core、可选扩展、文档还是下游产品。
