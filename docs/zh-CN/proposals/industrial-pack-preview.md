# 提案：OpenClaw.NET 行业包预览

本文档是一份提案。它不是承诺的路线图、发布承诺或对生态方向的所有权声明。

## 目的

探索 OpenClaw.NET 的可复用行业和边缘扩展，同时保持现有的自托管、NativeAOT 友好、供应商中立的项目方向。

此预览应帮助 .NET 开发者评估一个适用于制造业和边缘监控场景的轻量级、安全、可自托管的工业 AI 网关。

## 定位

OpenClaw.NET 可以为 .NET 开发者提供一个轻量级、安全、可自托管的工业 AI 网关。

行业包应通过可复用的适配器、示例和文档来扩展 OpenClaw.NET，而不应将特定产品的工业工作流移入 core。

## 初始目标

初始目标是制造业和边缘监控。

第一个参考实现设想是一个**机器监控 Agent**，能够基于模拟或清晰记录的输入，演示遥测采集、告警分类、维护工单创建和班次摘要生成。

## 候选适配器

候选适配器优先顺序：

1. MQTT 优先
2. Modbus 后续
3. OPC UA 后续
4. MES/ERP 连接器 后续

第一个适配器应优化为无需专有系统或供应商特定默认配置即可在 10 分钟内完成有价值的评估路径。

## 提议的分层架构

- OpenClaw.NET Core
- OpenClaw 行业包（Industrial Pack）
- 下游商业/产品解决方案

OpenClaw.NET Core 应聚焦于稳定的运行时契约、安全性、诊断和 NativeAOT 友好行为。

OpenClaw 行业包应包含可复用的工业抽象、适配器、示例、模板和文档。

下游商业或产品解决方案应负责客户特定工作流、专有 UX、工厂特定规则和供应商特定部署选择。

## 属于行业包的内容

- 协议抽象
- 遥测采集契约
- 模拟机器示例
- 告警分类示例
- 维护工单示例
- 班次摘要示例
- 诊断和可观测性示例
- 部署指南
- 中英文文档

## 不属于行业包的内容

- 客户特定的工厂逻辑
- 专有数字员工工作流
- 供应商专属默认配置
- 闭源商业 UX
- 产品特定的业务规则

## 治理

行业包必须保持供应商中立。

商业贡献者应在贡献直接支持公司或客户用例时进行披露。这有助于审查者评估变更是可复用基础设施、扩展工作、文档还是下游产品逻辑。

核心运行时变更需要核心维护者审查。影响网关安全态势、发布权限、包边界、默认 Provider 行为或公开路线图方向的变更也需要核心维护者审查。

区域维护者可以在其范围内指导行业包的文档、适配器、示例和社区反馈。
