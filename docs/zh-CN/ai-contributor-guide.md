# AI 贡献者指南

本文档为 AI 编程 Agent 和贡献者提供项目特定的上下文。

## 重要区域

### 运行时（Runtime）

运行时是核心 Agent 执行层。它应保持轻量级，并尽可能保持裁剪安全（trim-safe）。

### 网关（Gateway）

网关负责 HTTP、WebSocket、Webhook、认证、速率限制和可观测性相关事务。

### 插件兼容性

兼容性目标聚焦于主流工具插件路径，而非完全的上游扩展宿主对等。不支持的接口应快速失败并输出诊断信息。

### Semantic Kernel

SK 互操作是可选的，应保持与核心 NativeAOT 友好的运行时路径隔离。

### 实验分支

实验分支可能包含仅 JIT 或 MAF 特定的工作，不应自动视为默认架构。

## 推荐的 AI 贡献领域

- 测试
- 兼容性矩阵更新
- 文档改进
- 降低启动和组装复杂度的重构
- 可观测性改进

## 需要谨慎处理的领域

- 公开绑定的安全行为
- 插件兼容性声明
- NativeAOT 敏感的代码路径
- 新的强制性依赖
